Catch an idea üí°! To create an ordered version of the given data and compare elements with the same index. Count elements which are not equal.

Ok, what should I start from?

To get a new list, which is a sorted variant of some other list, use a built-in sorted() function. Ascending order by value is default so no need to change any other argument. 

data1 = sorted(data)
What to do next?

Now you need to iterate over both lists simultaneously, index by index. The perfect tool for this task is a built-in zip() function. It creates a generator, which yields values with the same index. Note, that you may use any number of iterable arguments, not only two. 

zip(data, sorted(data))

Not necessary to assign value to a variable if you are not going to use the variable more than once. Now iterate over pairs of values using for loop. 

for i, j in zip(data, sorted(data))
Explain, what should I do with the pairs?

You just need to compare them and get a logic value of their inequity. 

i != j for i, j in zip(data, sorted(data))
How to count number of unequal elements?

Python can interpret logic values as integers: True equals 1 and False equals 0. So each case of unequal elements gives you i != j => True => 1. You just need to use expression from the previous hint as argument of built-in sum() function. It accumulates 1-s, while 0-s have no effect. 

sum(i != j for i, j in zip(data, sorted(data)))
So, how my final solution should look like?

You just need to return a final value and wrap your code into function expression. Using type hints for input and output data are highly recommended.

Here is the whole solution:

def not_order(data: list[int]) -> int:

    return sum(i != j for i, j in zip(data, sorted(data)))

The solution is the same, but with using anonymous lambda function, operator standard module and built-in map() function. 

from operator import ne


not_order = lambda data: sum(map(ne, data, sorted(data)))
